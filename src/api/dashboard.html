⚡ Quick Guide

Broadcast Pool retains your transactions and broadcasts them when you choose: by time, block height, BTC price, or immediately.

Wallet → :50050 → Broadcast Pool → pending → Bitcoin network

Connect your wallet to the Electrum server URL shown in Settings — not the electrs indexer.

Behaviour depends on the nLockTime set when Sparrow signs the transaction.

Sparrow

  • nLockTime = 0 — TX stays pending. Schedule by date/time or BTC price (EUR/USD).
  • nLockTime = date (MTP) — follows global mode in Settings. Reschedule if MTP delays broadcast.
  • nLockTime = block height — broadcasts when the chain reaches that height.

Fiat price trigger only when nLockTime is disabled (0).

Use the same Electrum port (:50050). Cycle transactions arrive as pending for manual scheduling.

Liana

  • Refresh / cycle TX stays pending. Schedule the broadcast date (e.g. before timelock expiry).
  • Anti-fee-sniping uses a recent block height in nLockTime — scheduling still works.
  • Policy timelock (CSV in miniscript) is not read by Broadcast Pool.

Date/time only — no fiat price trigger for Liana.

Global settings apply to transactions with a real locktime in the TX itself (not manual pending ones).

Settings

  • Broadcast mode: Immediate, Scheduled, or By block height.
  • Wallet URL: Host:50050 from Settings. Disable Sparrow Tor proxy or txs bypass this pool.
  • BTC prices checked every 60s (Kraken, CoinGecko, Bitstamp).
